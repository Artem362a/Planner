# Напоминания

Личное напоминание = строка `notifications.reminders`: `text`, `remind_at`
(наивное локальное время, секунды обнулены), флаги доставки `sent`/`sent_at`,
ответ пользователя `ack`/`ack_at`, счётчик повторных доставок `repeat_count`,
повторяемость `recur_every`/`recur_unit`, происхождение `kind` +
`source_task_id`.

Жизненный цикл: **pending** (`sent=false`) → **доставлено** (`sent=true`) →
**ждёт ответа** (`ack is null`, бот повторяет доставку) → **отвечено**
(`ack='done'|'read'`). Повторяющееся после ответа перепланируется и снова
становится pending.

## Виды напоминаний

| kind | Откуда берётся |
|---|---|
| `manual` | создано руками: веб (колокольчик → ⏰) или бот (`/remind`) |
| `task` | автоматически из задачи дня с чекбоксом «напомнить» |

Плюс отдельные уведомления (не Reminder-строки): предупреждения о
**дедлайнах целей** — ежедневная проверка в час дайджеста.

## Настройки (аккаунт → «Напоминания», `PATCH /auth/reminder-settings`)

| Поле в `auth.users` | Смысл | Дефолт |
|---|---|---|
| `task_reminder_lead_min` | дефолт «за N минут» для чекбокса у задачи | 10 |
| `reminder_repeat_min` | повторная доставка без ответа через N минут (0 = выкл) | 30 |
| `reminder_repeat_max` | максимум повторов | 3 |
| `goal_deadline_days` | предупреждать о дедлайне цели за N дней (0 = выкл) | 3 |

## Доставка

Фоновый поток бота `_reminders_loop` (`bot.py`), тик ~30 секунд, два прохода:

1. **Первая доставка** (`_deliver_due_reminders`): `sent=false AND remind_at <= now`
   → `Notification`+`NotificationRecipient` (колокольчик) + сообщение в TG
   с кнопками; `sent=true` ставится до отправки в TG — при недоступном
   Telegram дубля не будет.
2. **Повторы и пропуски** (`_redeliver_unacked`): `sent=true AND ack is null`:
   - у повторяющегося наступил следующий цикл → перепланировать
     (`reminder_rules.reschedule_recurring`), даже если юзер так и не ответил;
   - иначе, если прошло `reminder_repeat_min` минут и `repeat_count <
     reminder_repeat_max` → повторное TG-сообщение «(повтор n/max)»
     (в колокольчик не дублируется — там уведомление и так непрочитано).

Кнопки под TG-сообщением: `✅ Сделано / 👀 Прочитано` (callback `rack:`)
и снуз `+10 мин / +1 час / +3 часа / Завтра утром` (callback `rsnz:`).

## Ответы (ack)

`POST /reminders/{id}/snooze` — как раньше; `POST /reminders/{id}/ack`
`{status: done|read}` (эквивалент кнопок бота):

- оба ответа останавливают повторные доставки (`ack`, `ack_at`);
- **done** у `kind='task'` дополнительно ставит задаче-источнику `status=1`;
- у повторяющегося любой ответ вызывает перепланирование на следующее
  срабатывание (сброс `sent/ack/repeat_count`);
- ack по ещё не сработавшему → 400; снуз сбрасывает `ack` и `repeat_count`.

## Повторяющиеся (`recur_every` × `recur_unit`: day|week|month)

Задаются при создании (веб-форма: «Повторять: дни/недели/месяцы» + N;
1 ≤ N ≤ 365, оба поля вместе). Следующее срабатывание считает
`reminder_rules.next_occurrence`: шаг от планового `remind_at` (не от момента
ответа), пропущенные циклы перешагиваются; для месяцев день зажимается концом
месяца (31 янв + 1 мес = 28/29 фев).

## Напоминания от задач дня (`kind='task'`)

У задачи с фиксированным временем в форме есть чекбокс «Напомнить за N минут»
(дефолт N — из настроек, поле `planning.day_tasks.remind_lead_min`,
null = не напоминать). Синхронизацию делает `_sync_task_reminder`
(`routers/day.py`) на create/update/reschedule:

- время/дата задачи изменились → напоминание переносится (и сбрасывается
  в pending);
- задача выполнена, чекбокс снят (`remind_lead_min: -1` в PATCH) или время
  напоминания уже в прошлом → напоминание удаляется;
- удаление задачи сносит напоминание по FK `ON DELETE CASCADE`;
- «Сделано» на таком напоминании в боте/через ack — задача выполнена.

## Дедлайны целей

Раз в сутки (вместе с дайджестом, `_send_goal_deadline_alerts`) по каждой
активной цели с `target_date`: за `goal_deadline_days` дней и в сам день
дедлайна — уведомление в колокольчик + `🎯` в TG. `goal_deadline_days=0`
отключает.

## Список и удаление

- Веб: дропдаун колокольчика — pending-напоминания с бейджами
  (`🔁 каждые N …` у повторяющихся, `📋` у задачных), снуз-чипы
  `+15м/+1ч/+1д`, удаление крестиком.
- Бот: `/reminders` — список активных с кнопками удаления.

Тесты: `backend/tests/test_reminders_v2.py` (правила повторяемости, ack,
синхронизация с задачами, настройки) и
`backend/tests/test_notifications.py` (базовые + снуз).
